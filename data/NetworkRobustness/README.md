# Network Robustness Dataset

This folder contains the experimental data associated with the network robustness evaluation of the control channel in the augmented reality teleoperation architecture for Xiaomi CyberDog.

The dataset is organized by experimental condition. Each condition contains the control commands generated from the Meta Quest 3 / Unity interface, the bridge-side communication log, the robot-side reception log, and a metadata file describing the network configuration and acquisition context.

The original baseline experiment is not duplicated in this folder. This directory focuses on additional network robustness conditions applied after the nominal control-channel evaluation.

## Dataset organization

```text
data/NetworkRobustness/
├── C1_bandwidth_reduction_10Mbps/
├── C2_packet_loss_3pct/
├── C3_delay_100ms/
├── C4_combined_5Mbps_100ms_3loss/
└── C5_distance_10m/
```

## Experimental conditions

```text
C1_bandwidth_reduction_10Mbps:
  Control-channel test with bandwidth limited to 10 Mbps.

C2_packet_loss_3pct:
  Control-channel test with induced packet loss of approximately 3%.

C3_delay_100ms:
  Control-channel test with induced network delay of approximately 100 ms.

C4_combined_5Mbps_100ms_3loss:
  Combined network degradation condition with bandwidth limitation, added delay, and packet loss.

C5_distance_10m:
  Distance-based condition evaluated at approximately 10 m between the operator-side system and the communication access point.
```

## Files per condition

Each experimental condition contains:

```text
quest_control_sent.csv
bridge_log.csv
robot_log.csv
metadata.yaml
```

## File description

```text
quest_control_sent.csv:
  Commands generated by the Meta Quest 3 / Unity AR interface.

bridge_log.csv:
  Bridge-side log associated with UDP reception, command parsing, intermediate processing, and ROS 2 publication.

robot_log.csv:
  Robot-side log associated with command reception or reconstructed control execution timing.

metadata.yaml:
  Description of the experimental condition, network configuration, acquisition context, and relevant notes.
```

## Evaluation scope

This dataset focuses on the control channel of the AR-ROS 2 teleoperation architecture. RGB and depth video robustness are not included in this folder unless explicitly stated in the corresponding metadata.

The logs allow the reconstruction of network robustness metrics such as command latency, packet integrity, valid command propagation, and temporal stability under different communication conditions.

## Data processing notes

The reported metrics should be obtained from valid operating windows of each experiment. During analysis, consistency checks may be applied to preserve only commands with coherent timing information and valid correspondence across the Unity, bridge, and robot-side logs.

Records with malformed timestamps, missing sequence correspondence, non-positive time intervals, duplicated identifiers, or isolated transient values outside the valid command-propagation window should be excluded from the statistical summaries. This avoids attributing acquisition artifacts or synchronization inconsistencies to the actual communication behavior of the teleoperation architecture.

## Reproducibility scope

The organization allows each network condition to be analyzed independently and compared against the nominal behavior reported in the main control-channel experiment.

The files are intended to support methodological transparency, independent verification of the reported metrics, and reproducibility of the network robustness evaluation.
